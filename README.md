# Лабораторна робота №3 з ООП (JavaScript + сторонні бібліотеки).  

**StudyBoard** – канбан-дошка для планування навчання.

**Мета** – реалізувати веб-додаток з використанням декількох JS-бібліотек, керуванням станом, локальним зберіганням і візуалізацією даних.

[Ретроспектива [UA]](./RETROSPECTIVE.md)  
[Канбан дошка](./screens/board.png) | [Статистика](./screens/stats.png) | [Налаштування](./screens/settings.png)

---

## Функціональність

- Канбан-дошка з трьома колонками: **"До виконання"**, **"В процесі"**, **"Виконано"**.
- Додавання задач із полями:
  - назва;
  - опис;
  - предмет (курс);
  - пріоритет.
- Перетягування задач між колонками (Drag & Drop).
- Видалення задач (з підтвердженням).
- Дизайн із кольоровим виділенням предметів (course badges + left border)
- Неонове світіння карток при наведенні (hover glow)
- Збереження стану дошки в `localStorage` (дані не губляться після перезавантаження).
- Сторінка **статистики** з двома графіками:
  - розподіл задач за статусами;
  - розподіл задач за курсами.
- Сторінка **налаштувань**:
  - вибір мови інтерфейсу (UA / EN);
  - очищення всіх задач.
- Підтримка двох мов інтерфейсу (i18n).

---

## Використані технології та бібліотеки

- **React + Vite** – основа фронтенду.
- **React Router** – навігація між сторінками (Board / Statistics / Settings).
- **Zustand** – глобальне керування станом (обʼєкт `Board`, задачі, курси).
- **localStorage** (через `StorageService`) – збереження дошки між сесіями.
- **@hello-pangea/dnd** – Drag & Drop задач між колонками.
- **Chart.js + react-chartjs-2** – побудова графіків на сторінці статистики.
- **i18next + react-i18next** – інтернаціоналізація (UA/EN).
- Власні допоміжні сервіси:
  - `Logger` – логування подій (завантаження, додавання, переміщення, видалення задач).
  - доменна модель: `Task`, `Course`, `Board`.

## Структура 

```text
src/
  domain/
    Task.js       // модель задачі
    Course.js     // модель курсу
    Board.js      // модель дошки (список задач + курсів)
  services/
    StorageService.js // робота з localStorage
    Logger.js         // просте логування в консоль
  state/
    useBoardStore.js  // Zustand store (board, addTask, moveTask, removeTask, getCourseById)
  components/
    common/
      Layout.jsx      // загальний каркас сторінки
      Header.jsx      // верхнє меню з навігацією
    board/
      BoardPage.jsx   // основна сторінка з канбан-дошкою
      Column.jsx      // окрема колонка (To Do / In Progress / Done)
      TaskCard.jsx    // картка задачі з кольорами та видаленням
      AddTaskForm.jsx // форма додавання задачі
    stats/
      StatsPage.jsx   // сторінка статистики
      Charts.jsx      // графіки (Chart.js)
    settings/
      SettingsPage.jsx // сторінка налаштувань
  i18n/
    i18n.js           // ініціалізація i18next
    uk.json           // українські тексти
    en.json           // англійські тексти
  App.jsx
  main.jsx
  App.css
  index.css
  ```

## Статистика

На сторінці статистики відображаються два графіки:

- Кругова діаграма: розподіл задач за статусами
- Стовпчикова діаграма: розподіл задач за предметами

Побудовано через Chart.js та react-chartjs-2.

## Запуск лабораторної роботи

У проєкті використовується дві окремі частини:

1. **Фронтенд-застосунок (React + Vite)** — знаходиться в директорії `studyboard/`
2. **Тестове середовище (Jest + Babel)** — знаходиться у корені репозиторію (`LAB3.SEM1`)

Це дозволяє розмежувати роботу інтерфейсу та модульне тестування бізнес-логіки.

---

## Як запустити застосунок 

1. Перейдіть у папку фронтенду

```bash
cd studyboard
```
2. Встановіть залежності
```bash
npm install
```
3. Запустіть застосунок
```bash
npm run dev
```

## Тести (Jest)

Тести запускаються з кореня лабораторної.

Вони покривають:

- об'єктно-орієнтовану модель Board (додавання, видалення, зміна статусу задач)

- логіку керування задачами

- захист від некоректних даних

- збереження цілісності даних

### Налаштування тестового середовища

У корені каталогу знаходяться:

**package.json** — залежності Jest та Babel

**babel.config.js** — конфігурація транспіляції

**jest.config.cjs** — налаштування Jest

**tests/board.test.js** — самі тести

### Запуск тестів

1. У корені LAB3.SEM1 виконайте:

```bash
npm install
```
```bash
npm test
```

Усі тести повинні завершитися статусом:
```bash
PASS  tests/board.test.js
```
